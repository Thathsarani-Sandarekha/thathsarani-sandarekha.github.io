What it does

  • Parses a PDF resume into structured JSON (name, contact, skills, experience).
  • Scores resume ↔ job description match with explanations.
  • Provides subscores (Skill Match, Role Relevance, Tech Stack, Industry Fit, Communication).
  • Runs embedding-based semantic similarity for an additional “grounded” signal.
  • Generates actionable feedback to improve both JD-fit and resume quality.

Why it matters

Recruiters need fast shortlisting, but keyword-only filters miss strong candidates. ResuMatch AI combines LLM reasoning with semantic similarity to produce a clear, explainable evaluation.

How it works

  1. PDF parsing: extract raw text using pdfplumber.
  2. Structured extraction: Groq LLaMA returns normalized JSON fields.
  3. Semantic similarity: SentenceTransformer embeddings compute JD–resume similarity.
  4. LLM scoring: LLaMA generates overall score + subscores + justification.
  5. Feedback: targeted improvement points for skills, clarity, and relevance.

Roadmap

  • Batch resume ranking (Top 5–10) using Vector DB + ranking logic
  • Export results to CSV
  • Chat-style assistant: ask questions about a resume (e.g., “Does this candidate know Docker?”)
  • Admin dashboard for multi-user recruiter workflows

Quick Facts

  • Category: Resume AI / Recruitment
  • App: Streamlit
  • LLM: Groq LLaMA 3.3 70B
  • Embeddings: all-MiniLM-L6-v2
  • Parsing: pdfplumber
  • Outputs: JSON + scores + feedback
