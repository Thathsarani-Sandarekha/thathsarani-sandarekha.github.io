Problem

Forecasting electricity demand is challenging due to seasonality, daily/weekly patterns, and long-range trends. This project focuses on building a strong ML baseline for hourly forecasting using engineered temporal features and robust time-series validation.

Approach

  • Feature engineering: hour, day-of-week, month, etc.
  • Lag features: supports yearly seasonality patterns.
  • TimeSeriesSplit: avoids data leakage and validates realistically.
  • Modeling: XGBoost training + evaluation + future forecasting.

Outputs

  • 1-year future hourly forecast.
  • Saved model (reload-ready) for reuse.
  • CSV export to power dashboards (Power BI).

What I built

  • Complete preprocessing pipeline for time-series ML.
  • Lag + temporal feature creation with clean train/test separation.
  • TimeSeriesSplit cross-validation workflow.
  • XGBoost training + evaluation + future forecast generation.
  • Model saving/loading + CSV output pipeline for BI dashboards.

Quick Facts

  • Category: Time Series Forecasting
  • Model: XGBoost
  • Validation: TimeSeriesSplit
  • Forecast Horizon: 1 Year
  • Dataset: PJME_hourly.csv (Kaggle – robikscube)
  • Deliverables: Saved model + CSV for Power BI

Tech Stack

Python, Pandas, NumPy, XGBoost, scikit-learn, Matplotlib, Seaborn
