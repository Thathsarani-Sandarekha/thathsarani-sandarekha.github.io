Mini RAG System — Movie Plots

A lightweight, memory-only RAG pipeline that answers questions about movie plots using embeddings + Chroma retrieval and a Groq LLM. Returns a structured JSON response: answer, contexts, and reasoning.

What it does

  • Loads and preprocesses a 500-row subset of Wikipedia Movie Plots.
  • Splits long plots into ~300-word chunks for retrieval efficiency.
  • Creates embeddings using sentence-transformers/all-MiniLM-L6-v2.
  • Stores vectors in an in-memory Chroma vector store.
  • Retrieves relevant plot snippets and queries a Groq LLM for answers.
  • Returns structured JSON: answer, contexts, reasoning.

Why this project matters

This is a clean demonstration of the real RAG workflow end-to-end: data → chunking → embeddings → retrieval → LLM grounding → structured output. It’s designed to be minimal and easy to extend into larger RAG systems later.

Pipeline summary

Dataset → Clean & Chunk (~300 words) → Embed (MiniLM) → Store (Chroma) → Retrieve Top-K → LLM Answer (Groq) → JSON Output (answer + contexts + reasoning)

What I built

  • Data preprocessing + chunking strategy for plot text
  • Embedding + storage + similarity retrieval pipeline
  • LLM prompting that restricts answers to retrieved contexts
  • Structured JSON schema for consistent downstream use
